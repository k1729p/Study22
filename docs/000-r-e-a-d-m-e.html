Research Kubernetes and Quarkus

The Kubernetes is a container-orchestration system for automating container deployment, scaling, and management.
The Quarkus is a Java framework tailored for deployment on Kubernetes.

The Panache is a Quarkus-specific library for the development of the Hibernate-based persistence layer. It is similar to Spring Data JPA.
For running the local Kubernetes cluster in Docker container it is used kind ('Kubernetes in Docker').
The Kubernetes images for Kafka, MongoDB, and PostgreSQL are from from Bitnami Images.

❶ Docker and Kubernetes Build

Action:

1. With batch file "01 Docker run Kafka MongoDB PostgreSQL.bat" create and run Docker containers
for Kafka broker, MongoDB database, and PostgreSQL database.
2. With batch file "02 Docker build sender and start.bat" build the Docker image and
start the container with the sender application.
3. With batch file "03 Docker build receiver and start Quarkus.bat" build the Docker image and
start the container with the receiver application. It builds the Quarkus native image.
4. With batch file "04 Kubernetes build.bat" create kind cluster, install Kafka, MongoDB, and PostgreSQL, and
load Docker images for the sender application and the receiver application.
5. With batch file "05 show Kubernetes logs.bat" show a log tail for a broker, databases and applications.
6. With batch file "06 CURL call Quarkus.bat" call the "read account by name" endpoint on Quarkus server.
7. With batch file "07 start Docker Quarkus.bat" restart the Docker container with the receiver application.
Before this starter batch execution the application should not run.

2.2. The Quarkus server with Kafka consumer and REST endpoints

The lists with Kafka records are consumed in the AccountConsumer class.
The payload with JSON content is deserialized and persisted as a Account entity in MongoDB database.

The GET endpoint in AccountResource class reads account by name from PostgreSQL database.
If entity is absent in PostgreSQL database, then it is read from MongoDB database and added as a new entity to PostgreSQL database.

For delivering the accounts to Kafka Broker it is responsible the sender application.
